# Product

## Register

product

## Users

Individual developers, students, small teams, security engineers, and quantitative
research learners who need to evaluate automated code repair, identify AI-specific
security risks, or study how a transparent model forms a bounded market opinion.
They may be learning, reviewing a pull request, preparing a release, triaging an
exposed application, or testing a research hypothesis against historical data.

## Product Purpose

Build one portfolio-grade, open-source monorepo with three evidence-first tools:
FreshPatch, a reproducible benchmark generator and execution-based evaluator for
code-repair systems; Sentinel, a defensive AI security checker for local source
code and public website surfaces; and SignalLab, a trainable, leakage-aware stock
research model that estimates relative performance without presenting personalized
investment advice. Success means a new user can run a meaningful local
demonstration quickly, inspect the evidence and limitations behind every output,
reproduce the result, and export artifacts suitable for technical review.

## Brand Personality

Precise, calm, trustworthy. The product should feel technically serious without becoming intimidating, and should communicate uncertainty honestly.

## Anti-references

Avoid neon “hacker movie” visuals, fear-based breach language, opaque risk scores,
noisy trading terminals, green/red certainty theater, generic SaaS card grids, and
decorative AI imagery. Do not imply that automated scanning replaces a professional
penetration test or that historical model performance guarantees future returns.

## Design Principles

1. Evidence before alarm: every result must show what was observed, why it matters, and how to verify it.
2. Progressive depth: make the first run approachable while preserving detailed technical context for experts.
3. Safe by construction: default to non-destructive, bounded checks and make active network behavior explicit.
4. Honest uncertainty: distinguish confirmed evidence, heuristics, and unavailable checks.
5. Reproducibility is a feature: commands, inputs, environments, and outputs should be easy to repeat.
6. Research is not advice: market outputs must name the prediction horizon,
   benchmark, data cutoff, held-out performance, uncertainty, and missing context.

## Accessibility & Inclusion

Target WCAG 2.2 AA. Support keyboard navigation, visible focus, reduced motion, semantic headings and landmarks, screen-reader labels, and color-independent severity indicators. Use plain language alongside security terminology and never rely on color alone to communicate status.
